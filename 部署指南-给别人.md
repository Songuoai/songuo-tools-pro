# 🚀 松果工具箱 - 快速部署指南

**适用对象：** 收到项目包后需要部署测试的人员  
**预计时间：** 10-15 分钟  
**难度等级：** ⭐⭐☆☆☆（简单）

---

## 📋 第一步：环境准备（5 分钟）

### 1.1 安装 Node.js

**为什么需要：** 项目基于 Next.js，需要 Node.js 运行环境

**下载地址：**
- 官网：https://nodejs.org/
- 推荐版本：Node.js 18.x 或 20.x（LTS 长期支持版）

**安装步骤：**

**Windows 用户：**
1. 下载 `.msi` 安装包
2. 双击运行，一直点"Next"
3. 安装完成后，打开命令提示符（Win+R → 输入 `cmd` → 回车）
4. 输入以下命令验证：
   ```bash
   node -v
   npm -v
   ```
5. 如果显示版本号（如 `v18.17.0` 和 `9.8.1`），说明安装成功 ✅

**Mac 用户：**
```bash
# 使用 Homebrew 安装
brew install node@18

# 验证安装
node -v
npm -v
```

---

### 1.2 解压项目包

1. 找到收到的压缩包 `松果工具箱-Pro-完整版.zip`
2. 右键 → 解压到当前文件夹
3. 解压后会得到一个名为 `松果工具箱-Pro-打包` 的文件夹
4. 建议重命名为 `松果工具箱-Pro`

---

## 📋 第二步：安装依赖（3-5 分钟）

### 2.1 打开命令行

**Windows：**
1. 进入解压后的文件夹 `松果工具箱-Pro-打包`
2. 在地址栏输入 `cmd`，回车
3. 或者：右键 → "在终端中打开" / "在 PowerShell 中打开"

**Mac/Linux：**
```bash
cd /path/to/松果工具箱-Pro-打包
```

### 2.2 安装项目依赖

在命令行中输入：

```bash
npm install
```

**等待安装完成：**
- 第一次安装需要 3-5 分钟
- 会下载约 200-300MB 的依赖包
- 完成后会显示 `found X packages in XXs`

**常见问题：**

**问题 1：下载速度慢**
```bash
# 使用淘宝镜像加速
npm config set registry https://registry.npmmirror.com
npm install
```

**问题 2：权限错误**
```bash
# Windows：以管理员身份运行 CMD
# Mac/Linux：在命令前加 sudo
sudo npm install
```

**问题 3：node_modules 已存在但报错**
```bash
# 删除 node_modules 重新安装
rm -rf node_modules package-lock.json
npm install
```

## 📋 第三步：配置环境变量（2 分钟）

### 3.1 创建 .env.local 文件

1. 在项目根目录找到 `.env.local.example` 文件
2. 复制一份，重命名为 `.env.local`

或者直接创建文件：

**Windows（记事本）：**
1. 右键 → 新建 → 文本文档
2. 命名为 `.env.local`（注意前面有个点）
3. 用记事本打开，粘贴以下内容

**Mac/Linux（终端）：**
```bash
cp .env.local.example .env.local
```

### 3.2 填写配置

打开 `.env.local` 文件，粘贴以下内容：

```env
# ========== Supabase 数据库配置 ==========
# 这些是示例配置，实际使用需要替换成你自己的 Supabase 项目配置

# Supabase 项目 URL（替换成你的）
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co

# Supabase 匿名密钥（替换成你的）
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key

# Supabase 服务密钥（替换成你的）
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key

# ========== 网站配置 ==========
# 本地测试用这个
NEXT_PUBLIC_SITE_URL=http://localhost:3000

# 生产环境改成你的域名
# NEXT_PUBLIC_SITE_URL=https://your-domain.com

# ========== 开发模式 ==========
# true = 开发模式（无需验证邮箱即可登录）
# false = 生产模式（需要邮箱验证）
NEXT_PUBLIC_DEV_MODE=true

# ========== 功能开关 ==========
# 是否使用模拟数据（true/false）
NEXT_PUBLIC_ENABLE_MOCK_DATA=false
```

**⚠️ 重要提示：**

上面的配置是示例，**实际使用需要替换成你自己的 Supabase 配置**！

**如何获取 Supabase 配置：**

1. 访问 https://supabase.com
2. 登录/注册账号
3. 创建新项目（免费）
4. 进入项目 → Settings → API
5. 复制以下三个值：
   - Project URL → `NEXT_PUBLIC_SUPABASE_URL`
   - anon public → `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - service_role → `SUPABASE_SERVICE_ROLE_KEY`

**如果不想配置 Supabase：**

可以开启模拟数据模式：

```env
NEXT_PUBLIC_ENABLE_MOCK_DATA=true
NEXT_PUBLIC_DEV_MODE=true
```

这样无需数据库也能测试基本功能！

## 📋 第六步：生产环境部署（可选）

### 6.1 Vercel 部署（推荐）

**优点：** 免费、自动部署、全球 CDN

**步骤：**

1. **访问 Vercel**
   ```
   https://vercel.com
   ```

2. **注册/登录**
   - 可用 GitHub/GitLab/Bitbucket 账号登录

3. **创建项目**
   - 点击 "New Project"
   - 选择 "Import Git Repository"
   - 导入你的 GitHub 仓库

   **或者拖拽部署：**
   - 点击 "Add New..." → "Project"
   - 选择 "Continue with [your source]"
   - 或者手动上传代码

4. **配置环境变量**
   - 在 Vercel 项目设置 → Environment Variables
   - 添加 `.env.local` 中的所有变量

5. **部署**
   - 点击 "Deploy"
   - 等待构建完成（2-5 分钟）
   - 获得部署域名：`https://your-project.vercel.app`

### 6.2 本地生产构建

**步骤：**

```bash
# 1. 构建
npm run build

# 2. 启动生产服务器
npm start

# 访问：http://localhost:3000
```

**注意：** 生产模式需要正确配置 Supabase 和关闭开发模式！

## 🐛 常见问题解决

### 问题 1：npm install 失败

**错误信息：** `npm ERR! code EACCES`

**解决方案：**
```bash
# Windows：以管理员身份运行 CMD
# Mac/Linux：
sudo npm install --unsafe-perm=true --allow-root
```

---

### 问题 2：端口被占用

**错误信息：** `Port 3000 is in use`

**解决方案：**
- Next.js 会自动尝试其他端口（3001, 3002...）
- 或者手动指定端口：
  ```bash
  npm run dev -- -p 3006
  ```

---

### 问题 3：页面显示"正在加载"

**可能原因：**
- API 配置错误
- Supabase 连接失败

**解决方案：**
1. 检查 `.env.local` 配置是否正确
2. 检查 Supabase 项目是否正常运行
3. 打开浏览器 F12 → Console 查看错误
4. 尝试开启模拟数据模式：
   ```env
   NEXT_PUBLIC_ENABLE_MOCK_DATA=true
   ```

---

### 问题 4：登录不了

**可能原因：**
- Supabase Auth 未配置
- 邮箱验证未关闭

**解决方案：**
1. 开启开发模式：
   ```env
   NEXT_PUBLIC_DEV_MODE=true
   ```
2. 或在 Supabase 控制台关闭邮箱验证

---

### 问题 5：构建失败

**错误信息：** `Type error` 或 `Build failed`

**解决方案：**
```bash
# 忽略 TypeScript 错误构建
# 修改 next.config.js，添加：
{
  typescript: {
    ignoreBuildErrors: true
  }
}
```
